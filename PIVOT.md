# PIVOT — 그룹 케미 매트릭스 (앱 단독, 화면 중심 한 장)

**2026-06-11** · 구판(공개 게시판·명예의 전당·닮은꼴)을 폐기하고 **"지인 그룹 N×N 케미 매트릭스"** 로 확정.
모르는 사람을 엮는 기능은 전부 버리고, **아는 사람들(단톡방·회식·MT·가족)의 궁합표**가 제품의 1차 기능.
기존 자산(궁합 엔진 6,300줄·pure 톤·코인·unlock·캡처 플로우)을 최대 재사용. 웹 결제 없음 — **결제는 전부 앱스토어 IAP**.

---

## 왜 이것인가 (기각 이력)

| 검토안 | 기각 사유 |
|---|---|
| 시술상담·안경 피팅 유틸 | 재사용되는 건 commodity 측정 레이어뿐. 해석 코퍼스 90% 사장 + 더 나은 대안 실존 |
| 웹 결제 퍼널 | 무명 사이트 웹 결제는 신뢰 없음 — 결제는 스토어 IAP 원칙 확정 |
| 장르 변경 + 유료 사주 | 4.3(b)는 카테고리 라벨이 아니라 기능을 봄. 사주 추가는 동질화 = 역효과 |
| 연예인 닮은꼴 | 부정경쟁방지법 타목(성명·초상 영업 사용) + metrics 자체가 생체인식 특징정보(민감정보). 사진 미저장도 회피 불가 |
| 일반인 닮은꼴 SNS | 공유 준거 없음 → 지위·공유가치 0. 콜드 스타트 + day-2 루프 부재 |
| 공개 게시판(구판) | 모르는 타인 탐색은 이 제품 본질이 아님 — 관심은 ①내 해석 ②공유 준거 비교 ③아는 사람과의 관계뿐 |

**살아남은 구조**: 초대가 기능이 아니라 전제인 콘텐츠 — "우리 팀 궁합표"는 혼자 존재할 수 없다.
성별 무관 우정·케미 프레임이라 단톡방에 던질 수 있고(데이팅 거부감·4.3(b) dating 회피),
궁합 엔진의 **pure 톤이 이미 비연애 페어용으로 구현돼 있음**.

## 코인 (확정)

| 행위 | 비용 |
|---|---|
| 방 만들기 · 멤버 스캔 · 매트릭스 보기 (셀 = 4단 케미 밴드 이모지) | **무료** |

## 인원 정책

**min 3 (2명은 기존 1:1 궁합으로) · 권장 4~8 · 하드캡 12 (66쌍).**
캡 근거는 연산이 아니라 UX — 폰 화면 매트릭스 가독성(12열 = 가로 스크롤 한계),
현장 스캔 시간(1인 30~60초 × 12 = 10분 = 인내 상한), 하위 밴드 박제 리스크(N² 증가).
13명+ 대형 그룹(신학기 반 등)은 수요 관측 후 랭킹 리스트 뷰로 별도 설계 (P3 이후).

## 매트릭스 화면 공통 UX

- 셀 = 4단 밴드 이모지 (💞 환상 케미 / 🔥 시너지 / 🤝 무난 / 🌧 보완 조합)
- **보는 사람 자신의 행을 항상 첫 행에 고정** (스크롤해도 상단에 붙음 — 엑셀 틀 고정처럼).
  사용자가 진짜 보려는 건 66칸이 아니라 "나 vs 누구" 11칸이기 때문
- **표 위에 요약 카드**: 🏆 베스트 페어(점수+한 줄 무료) · 😲 의외의 조합.
  표를 안 읽어도 하이라이트가 먼저 보이게
- 9명+ 는 가로 스크롤, 하위 밴드 문구는 "보완 조합" 프레임 고정

## 홈 재편 — "내 관상" 1급 객체화 · P1

현행 문제: 내 관상 설정이 촬영 → 관상 탭 리스트 → more 메뉴 → "내 관상 표시" 4단계.
내 관상은 피벗 후 방 참여(원탭)의 전제 조건이므로 홈의 1급 객체로 승격한다.
**버튼 4개(나/타인 × 카메라/앨범) 금지** — 홈은 "누구"만 묻고, "어디서"는 카메라 화면 안에서 해결.

```
내 관상 없을 때                      내 관상 있을 때
┌────────────────────────┐          ┌────────────────────────┐
│ [😶 내 관상 만들기]     │ 큰 CTA   │ ┌──┐ 닥터꾼 · 제왕형    │ ← 탭 = 내 리포트
│  셀카 한 장이면 끝      │          │ └──┘ 30대 · 둥근형      │
├────────────────────────┤          ├────────────────────────┤
│ 다른 사람 관상 보기     │ 보조     │ 다른 사람 관상 보기     │
│ [📷 촬영] [🖼 앨범]    │          │ [📷 촬영] [🖼 앨범]    │
└────────────────────────┘          └────────────────────────┘
```

- [내 관상 만들기] → **전면 카메라 즉시 오픈** (기본 경로 = 셀카, "촬영 → 내 관상 등장" 2스텝)
- **카메라 화면 좌하단에 앨범 아이콘** (카카오톡 프로필 변경·인스타 스토리 패턴)
  → 보정해 둔 내 사진을 앨범에서 선택해 내 관상으로 등록하는 경로. 선택 다이얼로그 없음
- 타인 촬영/앨범은 현행 2버튼 유지. 신규 화면 0개 — face_mesh_page 에 앨범 진입 + "내 관상 등록" 플래그
- 보정 사진 등록은 막지 않음(앨범 경로가 있는 한 불가능) — 케미 매칭도 그 metrics 로 돌아간다는 것만 전제
- 피벗 시 내 관상 카드 아래에 [그룹 케미 만들기] 가 자연스럽게 붙는다

## 모드 1 — 현장 모드 (설치 1대로 전원 참여) · P1

술자리·회식·MT에서 방장 폰 하나로 차례로 스캔. 설치 1명이 영업사원 N명을 만드는 보드게임 구조.

```
궁합 탭
└─ [그룹 케미 +]                ← compatibility_screen 에 엔트리 추가
   └─ 방 만들기: 모임명 ("마케팅팀 회식")
      └─ 멤버 추가 루프 (3~12명 · "4~8명이 가장 재밌어요" 가이드)
         ├─ 📷 카메라 → face_mesh_page 재사용
         ├─ 🖼 앨범   → album_capture_page 재사용
         └─ 성별·연령 확인 → info_confirm_screen 재사용(간소판)
      └─ [매트릭스 보기] (3명↑)
         ┌─ 마케팅팀 회식 ──────────────┐
         │      민지  준호  수아  부장님 │
         │ 민지   ·    💞    🤝    🌧   │
         │ 준호  💞    ·    🔥    🤝   │
         │ 수아  🤝    🔥    ·    💞   │
         │ 🏆 베스트: 민지××준호 91     │
         │    #호흡척척 "한 줄 풀이…"   │
         └──────────────────────────────┘
         └─ 페어 탭 → [점수·풀이 보기 1🪙]
                      → compatibility_detail_screen 재사용
```

## 모드 2 — 원격 모드 (딥링크 초대) · P2

```
방장: [방 만들기] → 내 관상 metrics 자동 합류 (없으면 셀카 스캔부터)
      → 카카오톡 초대 링크 facely.kr/g/{groupId}
초대받은 사람:
  ├─ 기설치 + 내 관상 있음 → 링크 탭 → [이 방에 참여] 한 번 → metrics 전송
  ├─ 기설치 + 내 관상 없음 → 셀카 스캔 → metrics 전송
  └─ 미설치 → /g/{id} 웹 초대장("9/12명 참여 · 당신 자리가 비어 있어요")
              → 스토어 → 설치 → 카톡에서 링크 재탭 → 셀카 → 전송
방 상태: 3명 차는 순간 부분 매트릭스 즉시 공개, 멤버 합류마다 행·열 성장
        (전원 대기 금지 — 안 들어오는 2명을 기다리면 방이 죽는다)
방장 [마감] (또는 48h 자동) → 🏆 베스트 페어 발표
  → /g/{id} 가 결과 쇼케이스로 전환 → 카톡 공유 → 본 사람이 자기 방 생성 → 루프
```

- **`/g/{id}` 는 한 라우트, 두 얼굴**: 마감 전 = 초대장 모드(참여 유도), 마감 후 = 결과 쇼케이스.
- **웹 공개 뷰는 이름 + 밴드 이모지만 기본** — 얼굴 썸네일 노출은 방장 옵트인.
  멤버가 동의한 건 그룹 안 공유지 전 세계 공개가 아니다.
- **react 는 엔진을 돌리지 않는다**: 궁합 엔진 22파일은 Flutter 에만 있고 shared(face_engine.js)
  엔 관상 엔진뿐. 앱이 계산한 밴드 매트릭스를 `groups.matrix_payload`(JSON) 에 저장하고
  react 는 그리기만 한다 — 궁합 엔진 shared 포팅 불필요.
- deferred deep link(설치 직후 자동 입장)는 외부 SDK 필요 → P2는 "카톡 재탭" 패턴으로 시작 (인프라 0).

## 부록 — mockup 생성용 ChatGPT prompt

사용법: 현재 앱 스크린샷 4장(홈·관상·궁합·설정 탭)을 **먼저 첨부**하고 아래 프롬프트를 붙여넣는다.
생성 결과가 디자인 제약을 어기면 "디자인 제약 위반. [불변 디자인 제약]을 다시 읽고 같은 화면만 재생성하라" 한 줄로 재시도.

```

대안을 제안하지 말고, 아래 명세 그대로 화면 mockup 이미지를 생성하라.

[확정된 기능 명세]
```
