# Part A — 확정 결정 (불변 스펙)

## A1. 왜 이것인가 (기각 이력)

| 검토안 | 기각 사유 |
|---|---|
| 시술상담·안경 피팅 유틸 | 재사용되는 건 commodity 측정 레이어뿐. 해석 코퍼스 90% 사장 + 더 나은 대안 실존 |
| 웹 결제 퍼널 | 무명 사이트 웹 결제는 신뢰 없음 — 결제는 스토어 IAP 원칙 확정 |
| 장르 변경 + 유료 사주 | 4.3(b)는 카테고리 라벨이 아니라 기능을 봄. 사주 추가는 동질화 = 역효과 |
| 연예인 닮은꼴 | 부정경쟁방지법 타목(성명·초상 영업 사용) + metrics 자체가 생체인식 특징정보(민감정보). 사진 미저장도 회피 불가 |
| 일반인 닮은꼴 SNS | 공유 준거 없음 → 지위·공유가치 0. 콜드 스타트 + day-2 루프 부재 |
| 공개 게시판(구판) | 모르는 타인 탐색은 이 제품 본질이 아님 — 관심은 ①내 해석 ②공유 준거 비교 ③아는 사람과의 관계뿐 |

**살아남은 구조**: 초대가 기능이 아니라 전제인 콘텐츠 — "우리 팀 궁합표"는 혼자 존재할 수 없다.
성별 무관 우정·케미 프레임이라 단톡방에 던질 수 있고(데이팅 거부감·4.3(b) dating 회피),
궁합 엔진의 **pure 톤이 이미 비연애 페어용으로 구현돼 있음**.

## A3. 인원 정책

**min 3 (2명은 기존 1:1 궁합으로) · 권장 4~8 · 하드캡 12 (66쌍).**
캡 근거는 연산이 아니라 UX — 폰 화면 매트릭스 가독성(12열 = 가로 스크롤 한계),
현장 스캔 시간(1인 30~60초 × 12 = 10분 = 인내 상한), 하위 밴드 박제 리스크(N² 증가).
13명+ 대형 그룹(신학기 반 등)은 수요 관측 후 랭킹 리스트 뷰로 별도 설계 (P4 이후).

## A4. 매트릭스 화면 공통 UX

- 셀 = 4단 밴드 이모지 (💞 환상 케미 / 🔥 시너지 / 🤝 무난 / 🌧 보완 조합)
- **보는 사람 자신의 행을 항상 첫 행에 고정** (스크롤해도 상단에 붙음 — 엑셀 틀 고정처럼).
  사용자가 진짜 보려는 건 66칸이 아니라 "나 vs 누구" 11칸이기 때문
- **표 위에 요약 카드**: 🏆 베스트 페어(점수+한 줄 무료) · 😲 의외의 조합.
  표를 안 읽어도 하이라이트가 먼저 보이게
- 9명+ 는 가로 스크롤, 하위 밴드 문구는 "보완 조합" 프레임 고정

## A5. 홈 개편 — 케미 중심 홈

현행 홈의 60%(수묵화 일러스트 + 타이틀)는 기능 0 인 브랜드 포스터. 피벗 후 홈은
**"내 방들이 살아 움직이는 곳"** — 방 카드가 day-2 리텐션 표면(푸시 탭 후 돌아올 자리)이 된다.

### 스플래시 (일러스트·타이틀의 이주)

- native splash **투명화** → 수묵화 + "관상은 과학이다." 타이틀은 **in-app 스플래시**로 이동.
- Android 12+ 는 OS 가 앱 아이콘을 시스템 스플래시에 강제로 그림 → `android_12.image` 에
  **transparent.png** 를 지정해 아이콘 자리를 비워 순수 백지로 만든다 (아이콘→수묵화 2단 갈아끼움 제거).
- in-app 스플래시는 **앱 부팅(auth 복원·Hive 로드, 실측 0.5~1초) 동안만** 노출. **인위적 지연 0초** —
  Apple HIG 는 launch 를 "즉시 사라지는 placeholder" 로 요구하며, 4.3(b) 재심사 중 감점 요인을 만들지 않는다.
- iOS 스플래시 캐시 좀비 주의: 교체 검증은 반드시 **앱 삭제 → 재부팅** 순서로.

## A6. 그룹 생성 UI — 검토안 3 + 채택

| 안 | 흐름 | 평가 |
|---|---|---|
| A. 풀스크린 위저드 | 이름 입력 → **현장/원격 모드 선택** → 멤버 수집, 3화면 | 명료하지만 생성까지 3화면 마찰. 치명 결함: 사용자에게 현장/원격 선택을 강요 — 사용자는 그 차이를 모르고, 알 필요도 없다 |
| **B. 바텀시트 퀵 생성 (채택)** | [＋] → 시트: 모임명 한 줄(제안 칩: 회식·MT·동아리·가족·스터디) → [만들기] → **즉시 방 화면**. 방 안에 [📷 직접 스캔] · [💬 카톡으로 초대] **상시 공존** | 생성 1스텝. 모드 선택 자체를 제거 |
| C. 캡처 우선 | [＋] → 바로 카메라, 연속 스캔 후 이름 붙이기 | 현장 최속이지만 원격 플로우가 절름발이, 카메라 권한 선요구, 이름 후행이 어색 |

**B 채택 사유:**

1. **현장/원격은 사용자가 미리 고르는 개념이 아니다.** 기본 시나리오가 혼합이다 — 회식 자리에서
   8명을 그 자리에서 스캔하고, 못 온 2명은 카톡으로 초대한다. 아래 A7 의 현장/원격은 **구현 경로**지
   **사용자 선택지**가 아니며, B 만이 이를 UI 로 보장한다 (A 는 잘못된 이분법을 첫 화면에 박는다).
2. **생성 마찰 최소** — 이름 한 줄이면 방이 생긴다. 빈 방도 가치가 있다: 초대장이 곧 콘텐츠다.
3. **C 의 장점을 포함한다** — 방 안 [📷 직접 스캔] = C 의 연속 스캔 루프(촬영 → 성별·연령 확인 →
   칩 추가 → "다음 사람" / "끝"). C 를 따로 만들 이유가 없다.
4. **바텀시트는 홈 컨텍스트를 유지한다** — 위저드처럼 "뒤로 가면 다 날아가는" 이탈 손실이 없다.

### 방 화면 (group_room_screen) 스펙

- 헤더: 모임명(방장 수정 가능) · 참여 n/N 진행바 · 마감 D-n
- 멤버 칩 그리드: 썸네일(없으면 이니셜) + 이름, 방장 👑, 빈 자리는 점선 "+" 칩
- 액션 (모든 멤버): [📷 직접 스캔] · [💬 카톡으로 초대] 나란히
- 방장 전용: [마감] — 3명 미만 비활성. 마감 시 🏆 발표 + `/g/{id}` 쇼케이스 전환
- 상태 3단: 모집 중 → 발표(마감) → 보관(30일 후)

### 원격 경로 (딥링크 초대)

```
방장: [방 만들기] → 내 관상 metrics 자동 합류 (없으면 셀카 스캔부터)
      → 카카오톡 초대 링크 facely.kr/g/{groupId}
초대받은 사람:
  ├─ 기설치 + 내 관상 있음 → 링크 탭 → [이 방에 참여] 한 번 → metrics 전송
  ├─ 기설치 + 내 관상 없음 → 셀카 스캔 → metrics 전송
  └─ 미설치 → /g/{id} 웹 초대장("9/12명 참여 · 당신 자리가 비어 있어요")
              → 스토어 → 설치 → 카톡에서 링크 재탭 → 셀카 → 전송
방 상태: 3명 차는 순간 부분 매트릭스 즉시 공개, 멤버 합류마다 행·열 성장
        (전원 대기 금지 — 안 들어오는 2명을 기다리면 방이 죽는다)
방장 [마감] (또는 48h 자동) → 🏆 베스트 페어 발표
  → /g/{id} 가 결과 쇼케이스로 전환 → 카톡 공유 → 본 사람이 자기 방 생성 → 루프
```

- **`/g/{id}` 는 한 라우트, 두 얼굴**: 마감 전 = 초대장 모드(참여 유도), 마감 후 = 결과 쇼케이스.
- **웹 공개 뷰는 이름 + 밴드 이모지만 기본** — 얼굴 썸네일 노출은 방장 옵트인.
  멤버가 동의한 건 그룹 안 공유지 전 세계 공개가 아니다.
- **react 는 엔진을 돌리지 않는다**: 궁합 엔진 22파일은 Flutter 에만 있고 shared(face_engine.js)
  엔 관상 엔진뿐. 앱이 계산한 밴드 매트릭스를 `groups.matrix_payload`(JSON) 에 저장하고
  react 는 그리기만 한다 — 궁합 엔진 shared 포팅 불필요.
- deferred deep link(설치 직후 자동 입장)는 외부 SDK 필요 → "카톡 재탭" 패턴으로 시작 (인프라 0).
